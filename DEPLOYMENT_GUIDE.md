# 🚀 Quick Start Guide - Get Your Portfolio Live in 5 Minutes

## Step-by-Step: Deploy to GitHub Pages (EASIEST)

### Step 1: Create GitHub Account (1 minute)
1. Go to https://github.com
2. Click "Sign up"
3. Enter email, password, username
4. Verify email
5. Done ✅

### Step 2: Create Repository (1 minute)
1. Click "+" icon (top right) → "New repository"
2. Repository name: `mounika-kuchana-portfolio`
3. Description: "Professional MLOps & DevOps Engineer Portfolio"
4. Select **"Public"**
5. ✅ Uncheck "Add a README file" (we have one)
6. Click "Create repository"

### Step 3: Upload Portfolio Files (2 minutes)

On the repository page:

1. Click "uploading an existing file"
2. Drag and drop these files:
   - `index.html`
   - `styles.css`
   - `script.js`
   - `README.md`
3. Click "Commit changes"

### Step 4: Enable GitHub Pages (1 minute)

1. Go to "Settings" (top right)
2. Scroll to "GitHub Pages"
3. Under "Build and deployment":
   - Source: Select "Deploy from a branch"
   - Branch: "main" 
   - Folder: "/ (root)"
   - Click "Save"

### Step 5: Get Your URL ✨

Your portfolio is LIVE at:
```
https://YOUR_USERNAME.github.io/mounika-kuchana-portfolio
```

**Example:**
```
https://mounika-kuchana.github.io/mounika-kuchana-portfolio
```

## 🚨 Troubleshooting

### "I don't see my site yet"
- Wait 2-3 minutes for GitHub to deploy
- Hard refresh browser (Ctrl+Shift+R on Windows)
- Check Settings → Pages to confirm it's deployed

### "Page looks broken"
- Make sure all 3 files are uploaded (html, css, js)
- Check file names are exact (case-sensitive)
- Hard refresh browser
